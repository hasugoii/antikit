# 🚀 AntiKit ワークフロースイート v1.0

**Vibe Codersのための完全なワークフローシステム** - アイデアがあればOK、AIがすべて対応。

> 💡 **哲学：** あなたは創造と決定を担当。AIがすべての技術を処理 - 必要だと知らなかったものも含めて。

---

## 📋 コマンド一覧 (20ワークフロー)

### 🌟 起動 & コンテキスト
| コマンド | 説明 | 対処するブラインドスポット |
|----------|------|--------------------------|
| `/init` | 完全な新規プロジェクト作成 | 環境変数, Git, コード品質ツール |
| `/recap` | コンテキスト復元 | コンテキスト回復 |
| `/save-brain` | 知識を保存 | API Docs, Changelog |
| `/config` | スキルとエージェントの設定 | リソース最適化 |

### 🎯 機能開発
| コマンド | 説明 | 対処するブラインドスポット |
|----------|------|--------------------------|
| `/brainstorm` | 計画前のアイデア探索 | 市場調査, MVP |
| `/plan` | 包括的な機能設計 | 認証, DB, チャート |
| `/visualize` | UI/UXデザイン | ローディング/エラー状態 |
| `/code` | 高品質なコード作成 | セキュリティ, バリデーション |

### ⚙️ 運用
| コマンド | 説明 |
|----------|------|
| `/run` | アプリを起動 |
| `/test` | ロジックを確認 |
| `/deploy` | 本番環境にデプロイ |

### 🔧 メンテナンス
| コマンド | 説明 |
|----------|------|
| `/debug` | バグ修正 |
| `/refactor` | コードのクリーンアップ |
| `/audit` | ヘルスチェック |
| `/rollback` | 以前のバージョンに戻す |

---

## 🎮 推奨ワークフロー

### 📦 新規プロジェクト
```
/init → /plan → /visualize → /code → /run → /test → /deploy → /save-brain
```

### 🌅 毎日の開始
```
/recap → /code → /run → /test → /save-brain
```

---

## 💡 Vibe Coderのヒント

1. **自然に話す** - AIが不足を聞いてくる
2. **間違いを恐れない** - `/rollback` がある
3. **1日の終わり: `/save-brain`** - 明日コンテキストを失わない
4. **定期的に `/audit`** - 予防は治療に勝る
