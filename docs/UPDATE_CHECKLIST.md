# AntiKit Release Checklist

> For contributors: follow this checklist when releasing a new AntiKit version.

## Pre-Release

- [ ] Bump `VERSION` file
- [ ] Update `manifest.json` version + verify agents/skills/workflows lists
- [ ] Update `CHANGELOG.md` with new entry
- [ ] Update `ARCHITECTURE.md` stats (agents, skills, workflows, scripts)
- [ ] Update 4 × `README*.md` badges (`vX.Y.Z-N_Skills`) + body text counts
- [ ] Verify all workflow files exist in 4 languages (en/vi/ja/zh)

## Skills

- [ ] New skills have `name`, `description` in YAML frontmatter
- [ ] T1-T3 skills have `connections` field (`uses` + `used_by`)
- [ ] Bidirectional: if A `uses` B, then B `used_by` A
- [ ] Update `ARCHITECTURE.md` Skill Tiers table if new T1-T3 skills added

## Workflows

- [ ] New workflows have `description` in YAML frontmatter
- [ ] All 4 language versions are consistent
- [ ] Workflows have "Auto-Save Lesson" section (📝)
- [ ] `/code` workflow has DoD Gate in Phase 7

## Agents

- [ ] New agents have `tags` in frontmatter for AGENT INDEX
- [ ] Agent-skill mappings are correct

## Installer Auto-Generated Sections

These are regenerated by `install.sh` / `install.ps1` when user runs `/ak-update`:

| Section | Source | Generated In |
|---------|--------|-------------|
| AGENT INDEX | Agent frontmatter `tags` | GEMINI.md |
| SKILL INDEX | Skill frontmatter `description` | GEMINI.md |

## Post-Release

- [ ] Test `install.sh` on clean machine (macOS/Linux)
- [ ] Test `install.ps1` on clean machine (Windows)
- [ ] Test `/ak-update` workflow
- [ ] Verify GEMINI.md regenerated correctly
- [ ] Push `.private/` submodule: `cd .private && git push`
- [ ] Push main repo: `git push`
- [ ] Tag release: `git tag v{VERSION} && git push --tags`
- [ ] Verify `brain.json` version + architecture stats match

## Sync antikit-web

- [ ] Update version in `antikit-web` (hero section, stats, etc.)
- [ ] Update agents/skills/workflows data files if counts changed
- [ ] Deploy to Cloudflare Pages
